commit b7868a471d1e9c2114073c43c71161e257fedf47
Author: John Kirkham <kirkhamj@janelia.hhmi.org>
Date:   Thu Jun 15 20:19:23 2017 -0400

    Construct a more natural environment to test conda
    
    Attempts to sort of emulate a `root`-like environment for `conda` to be
    in for testing. It is not truly the `root` environment though as it is
    still managed by `conda` from the actual `root` environment and this
    does show up in a couple of spots.
    
    Some of the things done include specifying a separate package cache,
    environment directory, and `.condarc` within the test environment.
    Further the actual `root` environment is scrubbed from the path. This is
    all done with a careful eye towards protecting a user that tries running
    these tests on their own machine so as to avoid messing up their `conda`
    install. Still users should approach this cautiously as this is very
    experimental at this point. In short this tries as much as possible to
    mimic a normal `root`-installed `conda` in as clean a way as possible.
    
    Adding this framework should make it easier to test things like
    activating and deactivating environments, create and removing
    environments, installing and uninstalling packages, etc. All in all this
    should allow us to have more confidence in `conda` packages before
    releasing them to `conda-forge`.

    Skip updating `conda` (on AppVeyor)
    
    For some strange reason `conda` 4.2.13 seems willing to do **anything**
    to upgrade itself. This includes downgrading from Python 3 to Python 2
    in order to get the latest `conda`. The result is we actually can't
    build and release `conda` for Python 3.5 and 3.6 currently.
    
    To fix this, we hack the AppVeyor script to disable the `conda` update
    step. Also configure `conda` to disable its auto-updating feature. So it
    won't try to update itself while doing something else. Hopefully this is
    sufficient to workaround the problem. A re-rendering will clear out this
    hack once we have completed the release.
